\begin{block}{Motivation}
Phishing is the dominant initial access vector in modern cyberattacks,
yet existing honeypots capture \textbf{only the credential submission stage}
— they do not track what attackers do after compromising an account.

\bigskip
\textbf{This work addresses three open gaps:}
\begin{enumerate}
    \item No multi-stage phishing honeypot links credential theft to
          post-compromise SSH behaviour in a single record.
    \item Existing systems lack automated, parameterised attacker
          simulation derived from real-world data.
    \item Behavioural profiling of attackers across the full lifecycle
          has not been attempted with multi-label ML classification.
\end{enumerate}
\end{block}

\begin{block}{System Architecture}
The platform integrates six components on a single Ubuntu host,
connected by a shared \textcolor{UVTLightBlue}{\textbf{session\_id}}
that links every event across all attack stages:

\bigskip
\begin{center}
\begin{tabular}{cl}
\toprule
\textbf{Stage} & \textbf{Component} \\
\midrule
1 & Phishing email (MailHog SMTP, port 1025) \\
2 & Flask decoy — login $\to$ MFA $\to$ success (port 5000) \\
3 & PostgreSQL credential store (port 5432) \\
4 & Cowrie SSH honeypot (port 2222, bare host) \\
5 & Filebeat $\to$ Elasticsearch (port 9200) \\
6 & Kibana dashboards (port 5601) \\
\bottomrule
\end{tabular}
\end{center}

\bigskip
Cowrie runs \textbf{directly on the host} (not Docker) to avoid SSH
port conflicts. Each of the four bot profiles occupies a dedicated
Docker subnet, producing genuinely distinct source IPs so the ML
classifier must learn behaviour, not addresses.
\end{block}

\begin{block}{PostgreSQL Schema}
Four tables store the complete attacker lifecycle:
\begin{itemize}
    \item \texttt{credentials} — 5 planted corporate identities
    \item \texttt{phishing\_sessions} — Flask interactions per session
    \item \texttt{ssh\_sessions} — Cowrie sessions and command logs
    \item \texttt{ml\_features} — extracted features + ground-truth labels
\end{itemize}

\bigskip
The \texttt{session\_id} foreign key present in all four tables is the
\textbf{core novel contribution}: it enables full end-to-end reconstruction
of an attacker's path from phishing page to SSH shell.
\end{block}

\begin{block}{Phishing Decoy — Three-Stage Flask Application}
The decoy impersonates a corporate login portal (\emph{CorpNet}) and
captures behavioural indicators at each stage:

\begin{enumerate}
    \item \textbf{Credential harvesting} — logs \texttt{time\_on\_page\_ms},
          \texttt{typing\_delay\_ms}, \texttt{field\_corrections}
    \item \textbf{MFA bypass} — generates a real 6-digit OTP via MailHog;
          bots read it programmatically from the MailHog REST API
    \item \textbf{Redirection} — fake dashboard closes the session record
\end{enumerate}
\end{block}

\begin{alertblock}{Attacker Bot Profiles}
Four profiles are parameterised from the \textbf{CyberLab Honeynet Dataset}
(293 real sessions) and \textbf{MITRE ATT\&CK} techniques T1078, T1059,
T1087, T1005:

\bigskip
\begin{center}
\begin{tabular}{lllr}
\toprule
\textbf{Profile} & \textbf{Login delay} & \textbf{MFA delay} & \textbf{Cmds} \\
\midrule
Scanner            & 0.1–0.5 s   & 0.1–0.3 s   & 4  \\
Cred. stuffer      & 0.5–2.0 s   & 0.2–0.5 s   & 4  \\
Human-like         & 3.0–12.0 s  & 8.0–30.0 s  & 11 \\
Sophisticated/APT  & 8.0–25.0 s  & 15.0–45.0 s & 16 \\
\bottomrule
\end{tabular}
\end{center}

\bigskip
Each cycle executes two phases: a \emph{victim bot} visits the Flask
decoy; an \emph{attacker bot} uses the stolen credentials to SSH into
Cowrie. Both phases write the same \texttt{session\_id}.
\end{alertblock}

\begin{block}{Cowrie SSH Honeypot}
Cowrie emulates a corporate Ubuntu server (\texttt{CORP-SRV-01}) with:
\begin{itemize}
    \item \textbf{5 planted user accounts} matching the phishing credentials
    \item \textbf{Fake filesystem} — \texttt{/etc/secrets.txt} with fictitious
          AWS keys and DB passwords; pre-populated \texttt{.bash\_history}
    \item \textbf{Wildcard authentication} — admits any credential to maximise
          session capture
    \item \textbf{JSON logging} — every command, timing, and session event
          shipped to Elasticsearch via Filebeat
\end{itemize}
\end{block}

\begin{block}{Machine Learning — Multi-Label Classification}
Two independent Random Forest classifiers share the same 6-feature vector:

\bigskip
\begin{center}
\begin{tabular}{ll}
\toprule
\textbf{Feature} & \textbf{Stage} \\
\midrule
\texttt{time\_on\_page\_ms}        & Phishing \\
\texttt{typing\_delay\_ms}         & Phishing \\
\texttt{field\_corrections}        & Phishing \\
\texttt{command\_count}            & SSH \\
\texttt{session\_duration\_s}      & SSH \\
\texttt{inter\_cmd\_delay\_avg\_s} & SSH \\
\bottomrule
\end{tabular}
\end{center}

\bigskip
\textbf{Label 1 — bot type:} scanner $|$ credential\_stuffer $|$
human\_like $|$ sophisticated

\textbf{Label 2 — attack stage:} cred\_submission $|$ ssh\_attack

\bigskip
Class imbalance is handled via \texttt{class\_weight=balanced}.
Evaluation uses 5-fold stratified cross-validation.
\end{block}

\bigskip
The perfect attack-stage score reflects the by-design separability:
phishing sessions have zero SSH features; SSH sessions have zero
phishing features. The macro F1 variance on bot type is driven by the
small sophisticated class sample and is expected to stabilise with
additional data collection.

\bigskip
\textbf{Top features (bot type RF):} \texttt{time\_on\_page\_ms} (25\%),
\texttt{typing\_delay\_ms} (23\%), \texttt{inter\_cmd\_delay\_avg\_s} (20\%).
\end{exampleblock}

\begin{alertblock}{Conclusions}
\begin{itemize}
    \item The first honeypot system to \textbf{link phishing credential
          theft to post-compromise SSH behaviour} via a shared session
          identifier.
    \item Four bot profiles derived from real honeynet data achieve
          \textbf{93\%+ classification accuracy} on bot type using only
          six behavioural features — no network-level features required.
    \item The multi-label architecture produces actionable intelligence:
          \emph{who} is attacking and \emph{what stage} they are in,
          simultaneously.
    \item Framework is fully automated, containerised, and reproducible
          for future research.
\end{itemize}
\end{alertblock}
